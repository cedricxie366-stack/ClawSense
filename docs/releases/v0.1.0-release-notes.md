# ClawSense v0.1.0

ClawSense 的首个公开可运行版本。

这个版本的目标不是“功能齐全”，而是把最关键的主链路跑通，并且让外部用户第一次接触项目时能够：

- 看懂它是什么
- 按文档完成部署
- 用旧 Android 手机完成配对
- 真正把音频、图片和心跳送到 OpenClaw 服务端

## 本版本已验证

下面这些能力已经在真实环境中跑通：

- OpenClaw 服务端插件加载
- 终端二维码生成
- Android 真机扫码配对
- 永久 `deviceSecret` 持久化
- 前台感知服务启动 / 停止
- 清晰可见的运行状态：
  - `未配对`
  - `启动中`
  - `运行中 · 完整模式`
  - `运行中 · 仅音频`
  - `运行中 · 仅图片`
  - `已停止`
  - `异常`
- 音频上传
- 图片上传
- 心跳上报
- 服务端 journal 持久化

## 安装

如果目标机器已经有 OpenClaw，可直接执行：

```bash
CLAWSENSE_NPM_SPEC="clawsense-openclaw-plugin@latest" bash install.sh
```

插件 npm 地址：

- [clawsense-openclaw-plugin](https://www.npmjs.com/package/clawsense-openclaw-plugin)

## 当前限制

这个版本仍然是可运行 MVP，不是最终成熟版。

- `2GB` 服务器在安装 OpenClaw 时仍可能 OOM
- 没有完整的设备管理页面
- 没有完整的上传失败重试队列
- 没有真正稳定的 STT 成果输出时，音频摘要可能退化为 `rms=...`
- 低内存服务器有时仍需要预打包运行时方案

## 后续方向

下一阶段优先做：

1. 接入真正稳定可用的 STT
2. 整理更稳的一键安装路径
3. 增加设备管理能力
4. 增加失败重试和上传队列
