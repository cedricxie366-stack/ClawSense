# ClawSense 小白部署与使用指南

这份文档面向两类人：

- 想把旧 Android 手机变成 ClawSense 感知节点的人
- 想把 ClawSense 接到自己 OpenClaw 服务端上的人

这不是“产品官网文案”，而是一份基于真实跑通过程整理出来的操作指南。  
能跑通的部分我会明确写“已验证”，还没做成正式发布态的部分我会明确写“当前限制”。

## 1. 当前已经验证通过的能力

下面这些链路已经在真实环境里跑通：

- OpenClaw 服务端公网可访问
- ClawSense 插件可加载
- Android 手机扫码配对成功
- 手机保存永久 `deviceSecret`
- App 重开后跳过重复配对
- 启动前台感知服务
- 服务状态明确显示：
  - `未配对`
  - `启动中`
  - `运行中 · 完整模式`
  - `运行中 · 仅音频`
  - `运行中 · 仅图片`
  - `已停止`
  - `异常`
- 音频 VAD 触发上传
- 首次图片上传会在启动后约 15 秒触发
- 后续图片上传为每 5 分钟 1 次
- 服务端可收到音频和图片
- 服务端会写入 ClawSense journal

## 2. 当前限制

这几个点现在要提前知道：

- 当前仓库更像“可运行的 MVP / 内测版”，还不是完全打磨好的公开发布版。
- 标准安装脚本 `install.sh` 在常规 Linux 环境可用，但在 **2GB 内存服务器** 上，安装 OpenClaw 时可能因为 `npm install` 被系统 OOM 杀掉。
- 如果服务端没有可用的 STT / Embedding / Vision 后端，ClawSense 仍然可以上传成功，但摘要可能降级成：
  - 音频：`rms=...`
  - 图片：回退描述
- 当前还没有完整的“设备管理后台页面”，设备查看主要通过 CLI。

## 3. 推荐环境

### 服务端推荐配置

- Linux 服务器或本地 Linux / macOS 电脑
- 推荐内存：`4GB` 及以上
- 如果只有 `2GB`：
  - 可能需要额外配置 `swap`
  - 仍可能在安装 OpenClaw 时失败
- 需要公网访问能力，或者至少手机能访问到服务端
- 需要放行 OpenClaw 端口
  - 本次验证使用的是 `18789`

### Android 客户端推荐配置

- Android 10 及以上
- 有摄像头和麦克风
- 能正常授予：
  - 相机权限
  - 麦克风权限
  - 通知权限

### 本次已验证环境

- OpenClaw：`2026.3.2`
- Android：Android 13
- 真机：OPPO `PDEM10`

## 4. 服务端部署

### 路线 A：标准环境（推荐）

适合：

- `4GB` 及以上内存的服务器
- 或本地电脑开发环境

步骤：

1. 安装 OpenClaw 稳定版  
   参考 OpenClaw 官方安装方式。

2. 让服务器拿到 ClawSense 插件来源  
   这里有 3 种方式，当前项目更常用的是前两种：

   方式 A：直接把整个仓库传到服务器
   - 适合当前这种“源码还在你自己手里、还没正式发布”的阶段
   - 常见做法：
     - 在服务器上 `git clone 你的仓库地址`
     - 或在本地把仓库打包后，用 `scp` 上传到服务器再解压
   - 这种方式下，服务器上最终会有这些文件：
     - `install.sh`
     - `index.ts`
     - `src/*`

   方式 B：给 `install.sh` 一个远程压缩包地址
   - 适合你把插件源码打成 `tar.gz`，放到对象存储、GitHub Release、静态文件服务器之后使用
   - 然后在服务器上执行：

```bash
CLAWSENSE_SOURCE_URL="https://your-host.example/clawsense-plugin.tar.gz" bash install.sh
```

   - `install.sh` 会自己先下载这个压缩包，再安装插件

   方式 C：插件已经发布到 npm
   - 适合后续正式发布阶段
   - 然后在服务器上执行：

```bash
CLAWSENSE_NPM_SPEC="clawsense-openclaw-plugin@latest" bash install.sh
```

   - `install.sh` 会直接走 `openclaw plugins install <npm-spec>`

   当前这份仓库还没有默认公开下载地址，所以文档里说“获取本仓库源码”，本质上就是：
   - 你先把源码仓库放到服务器
   - 或者给服务器一个可下载的插件压缩包 URL
   - 并不是服务器会自动知道该去哪里下载

3. 如果你走的是“方式 A：直接把仓库放到服务器”，就在仓库根目录执行：

```bash
bash install.sh
```

这个脚本会做几件事：

- 检测 OpenClaw 是否存在
- 必要时安装 OpenClaw `2026.3.2`
- 安装 ClawSense 插件
- 启用插件并加入 `plugins.allow`
- 重启 OpenClaw Gateway
- 直接在终端里执行 `openclaw clawsense pair`

### 路线 B：低内存服务器（2GB 机器）

适合：

- 阿里云轻量 / 低配 VPS
- 标准 `npm install` 会被 OOM 杀掉的环境

需要知道：

- 当前 `install.sh` 不保证在 2GB 机器上一次成功
- 这类环境更稳的做法是：
  - 先把 OpenClaw 运行时做成预打包运行目录
  - 再把 ClawSense 插件做成预装依赖包
  - 上传到服务器后手工解压部署

也就是说：

- **标准用户路径**：优先用 `install.sh`
- **低内存例外路径**：当前仍属于维护者支持的部署方式，不算最终对外一键发布形态

## 5. OpenClaw 端口与公网访问

手机要能访问到服务端，所以需要确认：

- Gateway 已启动
- 端口已放行
- 浏览器能从公网打开服务端地址

例如：

```bash
curl -i http://你的服务器IP:18789/
```

如果返回 OpenClaw Control 的 HTML 页面，说明公网访问基本正常。

## 6. 生成配对二维码

在服务端执行：

```bash
openclaw clawsense pair
```

它会在终端输出：

- 一个大号 ASCII 二维码
- 一段 JSON

JSON 里通常会包含：

- `token`
- `expiresAt`
- `ttlSeconds`

注意：

- `setup token` 只用于首次配对
- 不需要每次启动都重新扫码

## 7. Android App 首次使用

### 7.1 安装 App

当前 Android 工程在：

- [android](/Users/cedric/Documents/ClawSense/android)

用 Android Studio 打开这个目录即可。

### 7.2 第一次运行

首次进入 App 后：

1. 授予相机权限
2. 授予麦克风权限
3. 授予通知权限

如果某个权限没给全，App 仍可能进入降级模式：

- 只有麦克风：仅音频
- 只有相机：仅图片

### 7.3 配对

推荐方式：

1. 点 `扫码`
2. 扫服务器终端里的二维码
3. 完成配对

兜底方式：

如果二维码里的 Host 不对：

1. 手动填写 `Host`
2. 手动填写 `Setup Token`
3. 点击 `手动配对`

### 7.4 配对成功后会看到什么

配对成功后，页面会显示设备信息，例如：

- `设备 ID`
- `Host`
- `上传基址`
- `心跳间隔`

并且不会再停留在“首次配对”页面。

## 8. 启动感知服务后，页面状态是什么意思

这是当前 App 最重要的状态说明。

### `未配对`

表示：

- 手机还没有换到永久 `deviceSecret`
- 现在不能真正开始上传

### `启动中`

表示：

- 你刚点击了 `启动感知服务`
- App 已经开始切换到前台服务
- 一般会在 1 到 2 秒内进入运行状态

如果长时间停留在这里，通常要检查：

- 通知权限
- 前台服务是否真的起来
- 当前 App 是否是最新构建

### `运行中 · 完整模式`

表示：

- 音频 VAD 正在运行
- 定格拍照正在运行
- 心跳正在运行

这是最完整的正常状态。

### `运行中 · 仅音频`

表示：

- 麦克风在工作
- 相机没有参与
- 心跳仍然正常

常见原因：

- 相机权限没给
- 某些机型上相机前台服务受限制

### `运行中 · 仅图片`

表示：

- 相机在工作
- 麦克风没有参与
- 心跳仍然正常

常见原因：

- 麦克风权限没给

### `已停止`

表示：

- 前台服务已停止
- 音频采集停止
- 定格拍照停止
- 心跳停止

### `异常`

表示：

- 服务启动过，但某个关键步骤失败了

比如：

- 没有找到已配对设备
- 相机初始化失败

## 9. 真机验证时应该怎么判断“真的在工作”

### 音频

启动服务后：

- 对着手机说话 3 到 5 秒
- 服务端应该出现新的 `audio` 记录

当前 journal 文件通常在：

```bash
~/.openclaw/plugins/clawsense/state.json
```

### 图片

启动服务后：

- 第一次自动拍照大约在 `15` 秒后触发
- 之后每 `5` 分钟拍 1 次

如果手机日志里出现：

- `Still capture saved`
- `Uploading image`
- `Image upload succeeded`

就说明图片链路已经通了。

## 10. 常用服务端排查命令

### 查看设备列表

```bash
openclaw clawsense devices
```

### 重新生成配对二维码

```bash
openclaw clawsense pair
```

### 查看最近 journal

```bash
node - <<'NODE'
const fs = require('fs');
const path = require('path');
const file = path.join(process.env.HOME, '.openclaw', 'plugins', 'clawsense', 'state.json');
const state = JSON.parse(fs.readFileSync(file, 'utf8'));
const journal = Array.isArray(state.journal) ? state.journal : [];
console.log(JSON.stringify({
  count: journal.length,
  last: journal.slice(-10)
}, null, 2));
NODE
```

### 查看 OpenClaw Gateway 用户服务状态

```bash
systemctl --user status openclaw-gateway.service --no-pager -l
```

## 11. 常见问题

### 11.1 点了启动服务，但不知道是否真的成功

看 App 里的“服务状态”卡，不要只看按钮。

正确判断方式：

- `启动中`：正在切换前台服务
- `运行中 · 完整模式`：已经成功
- `已停止`：已经停掉

### 11.2 扫码成功，但配对失败

优先检查：

- 服务端二维码是否刚生成，`token` 是否还在 10 分钟有效期内
- Host 是否写对
- 手机是否能访问服务端公网地址

### 11.3 服务端安装 OpenClaw 时被系统杀掉

典型表现：

- `npm install` 出现 `Killed`
- `journalctl -k` 里能看到 OOM

优先建议：

- 换 `4GB` 及以上内存机器
- 或临时增加 `swap`
- `2GB` 机器当前不算稳定推荐配置

### 11.4 音频上传成功了，但摘要只有 `rms=...`

这说明：

- 录音和上传链路没问题
- 但服务端当前没有真正可用的语音转写后端

也就是说：

- **主链路已通**
- **智能摘要能力还没完全接上**

### 11.5 图片总是不上来

优先检查：

- 手机是否给了相机权限
- 服务是否处于运行中
- 日志里是否有：
  - `Still capture saved`
  - `Uploading image`
  - `Image upload succeeded`
